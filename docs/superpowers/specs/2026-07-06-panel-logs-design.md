# Panel Log 页设计文档

**日期：** 2026-07-06  
**范围：** Panel 前端新增 Log 页，实时展示 Panel 运行日志与请求日志。

## 目标

在 BSDock Panel Web 界面新增 `/logs` 页面，满足以下需求：

1. 显示 Panel 的**运行日志**（`panel.log`）和**请求日志**（`panel-requests.log`）。
2. 每类日志向前端推送**最近 200 条**的窗口。
3. 最新一行显示在列表最底部。
4. 默认跟随日志滚动更新；用户手动滚动视窗时取消底部锁定；重新滚动到底部后再次触发锁定。
5. 提供“回到底部”按钮。
6. 根据日志级别（INFO / WARN / ERROR 等）进行颜色区分。
7. 支持在运行日志与请求日志之间切换筛选。

## 架构

采用**后端环形缓冲 + 独立 WebSocket 通道**方案：

- 后端在内存中维护两个固定容量为 200 的环形缓冲区，分别缓存运行日志和请求日志的最新条目。
- 新增独立 WebSocket 端点 `/ws/logs`，用于向前端实时推送日志。
- 前端通过 WebSocket 订阅日志流，首次连接时接收历史 200 条，之后逐条接收新增日志。
- 前端本地维护当前筛选类型的日志数组，并实现滚动跟随/锁定逻辑。

## 后端设计

### 1. 日志采集

- **运行日志：** 将 Panel 的全局 `log` 输出重定向到一个 `io.MultiWriter`，其中一个 Writer 写入原有的 `panel.log` 滚动文件，另一个 Writer 把格式化后的日志行写入运行日志环形缓冲区。
- **请求日志：** 请求中间件 `RequestLoggingMiddleware` 同样把日志行同时写入 `panel-requests.log` 滚动文件和请求日志环形缓冲区。

### 2. 环形缓冲区

- 容量固定为 200。
- 支持线程安全写入。
- 提供 `Snapshot()` 返回当前缓冲区中所有条目（按时间顺序）。
- 提供 `Subscribe(ch)` / `Unsubscribe(ch)`，新日志写入时广播到所有订阅通道。

### 3. WebSocket 端点 `/ws/logs`

- 使用与现有 `/ws` 相同的 JWT 认证方式（查询参数 `token`）。
- 连接建立后：
  1. 根据 token 解析用户，验证通过后发送当前筛选类型的最近 200 条日志（JSON 数组）。
  2. 订阅对应日志类型的实时通道。
  3. 后续每产生一条新日志，立即推送给客户端。
- 客户端通过 WebSocket 消息中的 `type` 字段区分运行日志（`runtime`）和请求日志（`request`）。
- 前端可在连接后发送消息切换订阅类型（例如 `{ "action": "subscribe", "source": "request" }`），后端返回对应类型的历史 200 条并切换推送。

### 4. 日志条目结构

```json
{
  "timestamp": "2026-07-06T14:32:01+08:00",
  "level": "INFO",
  "source": "runtime",
  "message": "panel listening on :8080"
}
```

- `timestamp`：日志生成时间（RFC3339）。
- `level`：通过正则或前缀解析，无法识别时默认为 `INFO`。
- `source`：`runtime` 或 `request`。
- `message`：原始日志内容。

## 前端设计

### 1. 路由

- 新增 `/logs` 文件路由：`web/src/routes/logs/index.tsx`。
- 在 `AppSidebar` 的 `navItems` 中新增 `Logs` 导航项，图标使用 `ScrollText`。

### 2. 状态管理

新增 hook `useLogs(source: 'runtime' | 'request')`：

- 建立到 `/ws/logs?token=...` 的 WebSocket 连接。
- 接收初始历史日志数组，设置到本地状态。
- 接收实时日志条目，追加到数组末尾；数组长度超过 200 时从头部移除。
- 处理连接断开自动重连（指数退避）。
- 当 `source` 切换时，向后端发送订阅切换消息，并清空当前日志数组等待新数据。

### 3. Log 页面组件

页面结构：

```
+--------------------------------------------------+
| Logs                              [runtime] [request] |
+--------------------------------------------------+
|                                                  |
|  2026-07-06 14:32:01 [INFO] panel listening...   |
|  2026-07-06 14:32:02 [INFO] node foo online...   |
|  ...                                             |
|                                                  |
+--------------------------------------------------+
|              [ 跳到底部 ]                        |
+--------------------------------------------------+
```

- 顶部使用 Tabs 切换运行日志 / 请求日志。
- 中部为可滚动日志列表容器，固定高度（`flex-1` 或 `h-[calc(100vh-...)]`）。
- 每条日志为等宽字体 `<pre>` 或 `div`，按级别着色：
  - `INFO`：默认文本色
  - `WARN`：琥珀色
  - `ERROR`：玫瑰色
  - `DEBUG`：灰色
- 列表底部保持一条透明占位元素，用于 `scrollIntoView` 滚动到底部。
- 当新日志到达且处于“锁定”状态时，自动滚动到底部。

### 4. 滚动锁定逻辑

- 使用 `useRef` 持有日志容器 DOM。
- 使用 `useState` 维护 `isLocked`（默认 `true`）。
- 监听容器 `scroll` 事件：
  - 若用户滚动导致底部不在视口内（`scrollTop + clientHeight < scrollHeight - threshold`），设置 `isLocked = false`。
  - 若用户滚动到底部附近（`scrollTop + clientHeight >= scrollHeight - threshold`），设置 `isLocked = true`。
- 当 `isLocked` 为 `true` 且日志数组更新时，调用 `scrollToBottom()`。
- “回到底部”按钮：点击后滚动到底部并设置 `isLocked = true`。

### 5. 样式

- 延续现有暗色主题（`bg-[#080A0F]`、文本 `#E8EBF0`、强调色 `#00F0FF`）。
- 日志容器使用玻璃态卡片（`glass` class）。
- 时间戳使用次要文本色 `#8B95A8`。
- 级别标签使用小写等宽字体并着色。

## 数据流

1. 用户打开 `/logs` 页面。
2. `useLogs('runtime')` 建立 WebSocket 连接。
3. 后端验证 JWT，发送运行日志历史 200 条。
4. 前端渲染日志并滚动到底部。
5. Panel 产生新日志 → 后端写入环形缓冲区并广播 → WebSocket 推送到前端。
6. 前端追加日志；若 `isLocked` 则自动滚动到底部。
7. 用户滚动列表 → `isLocked` 变为 `false` → 新日志不再自动滚动。
8. 用户点击“回到底部” → `isLocked` 变为 `true` 并滚动到底部。

## 测试策略

- **后端单元测试：**
  - 环形缓冲区写入/读取/容量限制。
  - 日志级别解析（INFO/WARN/ERROR/DEBUG）。
  - `/ws/logs` 认证失败返回 401。
  - 订阅切换消息处理。
- **前端单元测试：**
  - `useLogs` hook 接收初始历史日志后正确设置状态。
  - 滚动锁定逻辑：用户滚动时 `isLocked` 变化。
  - 日志级别着色组件渲染正确颜色。
- **E2E 测试：**
  - 登录后访问 `/logs` 能看到日志内容。
  - 切换请求日志 tab 显示请求日志。
  - 滚动后“回到底部”按钮出现并生效。

## 风险评估

- **内存占用：** 两个 200 条日志缓冲，单条约 200 字节，合计约 80KB，可忽略。
- **WebSocket 并发：** 日志 WebSocket 仅内部管理员使用，并发量低。
- **向后兼容：** 不影响现有 `/ws` 和 agent WebSocket。
